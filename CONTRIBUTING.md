# Contributing to GitHub SEO Repository Template

Thank you for your interest in improving this template! This document provides guidelines for contributing.

## How to Contribute

### Reporting Issues

- Use the [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md) template for bugs
- Use the [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md) template for new ideas
- Check existing issues before creating a new one

### Submitting Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-improvement`)
3. Make your changes following the template's structure
4. Ensure README.md maintains SEO best practices
5. Commit with clear messages (`git commit -m 'Add: description'`)
6. Push to your branch (`git push origin feature/amazing-improvement`)
7. Open a Pull Request using the [PR template](.github/PULL_REQUEST_TEMPLATE.md)

## Contribution Guidelines

### README Updates

- Maintain the required structure (H1, H2, H3 hierarchy)
- Keep the first paragraph as WHAT + WHO + PLATFORM
- Use clear, literal language over clever phrasing
- Include practical examples when possible

### Workflow Updates

- Keep workflows minimal and focused
- Document what each workflow does
- Make workflows optional/opt-in where possible
- Test workflows before submitting

### Documentation

- Write for developers with zero context
- Assume readers arrive from search engines
- Include code examples where applicable
- Link to official documentation

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help maintain the template's clarity and simplicity

## Questions?

Open an issue with the `question` label for any clarifications.
